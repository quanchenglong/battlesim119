# 方案四：静态网站托管详细教程

本教程将详细说明如何使用GitHub Pages、Vercel和Netlify三种方式部署你的战斗模拟器项目。

---

## 方法一：GitHub Pages（完全免费，推荐）

### 前置准备
1. 注册GitHub账号：https://github.com/
2. 安装Git（如果未安装）：https://git-scm.com/downloads

### 第一步：安装Git并配置

#### 1.1 检查Git是否已安装
打开命令提示符（CMD）或PowerShell，输入：
```bash
git --version
```
如果显示版本号，说明已安装；如果提示"不是内部或外部命令"，需要先安装Git。

#### 1.2 配置Git用户信息（首次使用需要）
```bash
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱@example.com"
```

### 第二步：在GitHub上创建仓库

#### 2.1 登录GitHub
访问 https://github.com/ 并登录

#### 2.2 创建新仓库
1. 点击右上角的 **"+"** 号
2. 选择 **"New repository"**
3. 填写仓库信息：
   - **Repository name**: `battle-simulator`（或任意名称）
   - **Description**: `战斗模拟器 - 战斗计算工具`（可选）
   - **Visibility**: 选择 **Public**（公开，GitHub Pages免费版需要公开仓库）
   - **不要勾选** "Initialize this repository with a README"
4. 点击 **"Create repository"**

### 第三步：初始化本地Git仓库并上传代码

#### 3.1 在项目目录打开终端
在项目文件夹 `C:\Users\fukai\testc116` 中：
- **方法A**：按住Shift键，右键点击文件夹空白处，选择"在此处打开PowerShell窗口"
- **方法B**：在文件资源管理器地址栏输入 `cmd` 或 `powershell` 后回车

#### 3.2 初始化Git仓库
```bash
git init
```

#### 3.3 添加所有文件到Git
```bash
git add .
```
（注意：`.` 表示当前目录所有文件）

#### 3.4 提交文件
```bash
git commit -m "Initial commit: 战斗模拟器项目"
```

#### 3.5 连接到GitHub仓库
```bash
git remote add origin https://github.com/你的用户名/仓库名.git
```
**示例**：如果你的GitHub用户名是 `zhangsan`，仓库名是 `battle-simulator`，则：
```bash
git remote add origin https://github.com/zhangsan/battle-simulator.git
```

#### 3.6 推送代码到GitHub
```bash
git branch -M main
git push -u origin main
```

**注意**：首次推送会要求登录GitHub，按提示操作：
- 如果提示输入用户名和密码，用户名是GitHub用户名
- 密码需要使用Personal Access Token（见下方说明）

#### 3.7 创建Personal Access Token（如果需要）
1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 填写信息：
   - **Note**: `Git Push Token`
   - **Expiration**: 选择有效期（建议90天或更长）
   - **Select scopes**: 勾选 `repo`（全部权限）
4. 点击 **"Generate token"**
5. **重要**：复制生成的token（只显示一次），在推送时作为密码使用

### 第四步：启用GitHub Pages

#### 4.1 进入仓库设置
1. 在GitHub仓库页面，点击 **"Settings"**（设置）标签
2. 在左侧菜单找到 **"Pages"**（页面）

#### 4.2 配置Pages
1. 在 **"Source"** 部分：
   - 选择 **"Deploy from a branch"**
   - **Branch**: 选择 `main`
   - **Folder**: 选择 `/ (root)`
2. 点击 **"Save"**（保存）

#### 4.3 等待部署完成
- 通常1-2分钟后，页面会显示：
  ```
  Your site is live at https://你的用户名.github.io/仓库名/
  ```

### 第五步：访问你的网站

访问地址格式：
```
https://你的用户名.github.io/仓库名/
```

**示例**：
- 用户名：`zhangsan`
- 仓库名：`battle-simulator`
- 访问地址：`https://zhangsan.github.io/battle-simulator/`

### 后续更新代码

每次修改代码后，执行以下命令更新网站：
```bash
git add .
git commit -m "更新说明"
git push
```
等待1-2分钟后，网站会自动更新。

## 方法二：Vercel（免费，自动部署）

### 前置准备
1. 注册Vercel账号：https://vercel.com/
2. 需要先完成GitHub部署（方法一的前三步）

### 第一步：登录Vercel
1. 访问 https://vercel.com/
2. 点击 **"Sign Up"** 或 **"Log In"**
3. 选择 **"Continue with GitHub"**（使用GitHub账号登录）

### 第二步：导入项目
1. 登录后，点击 **"Add New..."** → **"Project"**
2. 在项目列表中找到你的仓库（如 `battle-simulator`）
3. 点击 **"Import"**

### 第三步：配置项目
1. **Project Name**: 可以修改或保持默认
2. **Framework Preset**: 选择 **"Other"** 或 **"Other"**
3. **Root Directory**: 保持默认 `./`
4. **Build Command**: 留空（纯HTML项目不需要构建）
5. **Output Directory**: 留空或填写 `./`
6. 点击 **"Deploy"**

### 第四步：等待部署
- 通常30秒-1分钟完成
- 部署完成后会显示访问链接，格式：
  ```
  https://项目名.vercel.app
  ```

### 第五步：自定义域名（可选）
1. 在项目设置中，点击 **"Settings"** → **"Domains"**
2. 输入你的域名（如果有）
3. 按照提示配置DNS

### 自动更新
- 每次向GitHub推送代码，Vercel会自动重新部署
- 无需手动操作

## 方法三：Netlify（免费，拖拽部署）

### 第一步：注册Netlify
1. 访问 https://www.netlify.com/
2. 点击 **"Sign up"**
3. 选择 **"Sign up with GitHub"**（使用GitHub账号）

### 方法A：拖拽部署（最简单）

#### 1. 准备项目文件
确保项目文件夹中只有需要的文件（`index.html`等）

#### 2. 拖拽部署
1. 登录Netlify后，进入控制台
2. 找到 **"Sites"** 区域
3. 将整个项目文件夹拖拽到 **"Want to deploy a new site without connecting to Git? Drag and drop your site output folder here"** 区域
4. 等待上传和部署完成（通常1-2分钟）

#### 3. 获取访问链接
部署完成后会显示：
```
https://随机名称.netlify.app
```

### 方法B：GitHub连接部署（推荐）

#### 1. 连接GitHub仓库
1. 点击 **"Add new site"** → **"Import an existing project"**
2. 选择 **"Deploy with GitHub"**
3. 授权Netlify访问GitHub
4. 选择你的仓库

#### 2. 配置部署
1. **Branch to deploy**: `main`
2. **Build command**: 留空
3. **Publish directory**: 留空或填写 `./`
4. 点击 **"Deploy site"**

#### 3. 等待部署
- 通常1-2分钟完成
- 会显示访问链接

### 自定义域名
1. 在站点设置中，点击 **"Domain settings"**
2. 点击 **"Add custom domain"**
3. 输入域名并配置DNS

## 常见问题解决

### 问题1：Git推送时提示认证失败
**解决**：使用Personal Access Token代替密码

### 问题2：GitHub Pages显示404
**解决**：
1. 检查仓库是否为Public
2. 检查Settings → Pages配置是否正确
3. 等待几分钟后刷新

### 问题3：网站显示空白
**解决**：
1. 检查`index.html`是否在根目录
2. 检查浏览器控制台是否有错误
3. 确保所有外部资源（如CDN链接）可访问

### 问题4：更新后网站未刷新
**解决**：
1. 清除浏览器缓存（Ctrl+F5强制刷新）
2. 等待1-2分钟让CDN更新

## 推荐流程

**新手推荐**：
1. 先使用 **GitHub Pages**（最稳定，文档最全）
2. 熟悉后再尝试 **Vercel**（部署最快）

**快速演示**：
- 使用 **Netlify拖拽部署**（无需Git，最快）

**长期使用**：
- 使用 **GitHub Pages** 或 **Vercel**（自动更新，更省心）
