# 获取外网访问链接 - 完整指南

## 方法一：GitHub Pages（免费，推荐）

### 如果你已经启用了GitHub Pages

你的外网访问链接格式为：
```
https://你的GitHub用户名.github.io/仓库名/
```

**示例**：
- GitHub用户名：`zhangsan`
- 仓库名：`battle-simulator`
- 访问链接：`https://zhangsan.github.io/battle-simulator/`

### 如何查找你的GitHub Pages链接

1. 进入你的GitHub仓库页面
2. 点击 **Settings**（设置）
3. 在左侧菜单点击 **Pages**
4. 在页面顶部会显示：
   ```
   Your site is live at https://你的用户名.github.io/仓库名/
   ```
5. 点击这个链接即可访问

### 如果还没有启用GitHub Pages

按照以下步骤操作：

#### 步骤1：确保代码已上传到GitHub

**方法A：网页上传（最简单）**
1. 进入GitHub仓库页面
2. 点击 **"Add file"** → **"Upload files"**
3. 拖拽 `index.html` 到页面
4. 点击 **"Commit changes"**

**方法B：使用Git命令行**
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

#### 步骤2：启用GitHub Pages

1. 进入仓库 **Settings** → **Pages**
2. **Source**: 选择 "Deploy from a branch"
3. **Branch**: 选择 `main`（或 `master`）
4. **Folder**: 选择 `/ (root)`
5. 点击 **Save**

#### 步骤3：等待部署

- 等待1-2分钟
- 页面会显示你的访问链接
- 点击链接即可访问

## 方法二：Vercel（免费，部署更快）

### 快速部署步骤

1. **访问Vercel**：https://vercel.com/
2. **登录**：点击 "Sign Up"，选择 "Continue with GitHub"
3. **导入项目**：
   - 点击 "Add New..." → "Project"
   - 找到你的GitHub仓库
   - 点击 "Import"
4. **配置项目**：
   - Framework Preset: 选择 "Other"
   - 其他保持默认
   - 点击 "Deploy"
5. **获取链接**：
   - 等待30秒-1分钟
   - 部署完成后会显示访问链接
   - 格式：`https://项目名.vercel.app`

### 优势
- ✅ 部署速度快（30秒）
- ✅ 自动HTTPS
- ✅ 自动更新（每次GitHub推送自动重新部署）
- ✅ 全球CDN加速

## 方法三：Netlify（免费，支持拖拽）

### 方法A：拖拽部署（最简单）

1. **访问Netlify**：https://www.netlify.com/
2. **注册登录**：使用GitHub账号登录
3. **拖拽部署**：
   - 将项目文件夹拖拽到Netlify控制台
   - 等待上传和部署（1-2分钟）
4. **获取链接**：
   - 部署完成后显示访问链接
   - 格式：`https://随机名称.netlify.app`

### 方法B：GitHub连接部署

1. 登录Netlify
2. 点击 "Add new site" → "Import an existing project"
3. 选择 "Deploy with GitHub"
4. 选择你的仓库
5. 点击 "Deploy site"
6. 等待部署完成，获取链接

## 方法四：使用ngrok（临时外网访问）

如果你只想临时让外网访问本地服务器：

### 步骤

1. **下载ngrok**：https://ngrok.com/download
2. **注册账号**：https://dashboard.ngrok.com/signup
3. **获取authtoken**：
   - 登录后，在Dashboard中复制authtoken
4. **配置ngrok**：
   ```bash
   ngrok config add-authtoken 你的token
   ```
5. **启动本地服务器**：
   ```bash
   python -m http.server 8000
   ```
6. **启动ngrok**（新开一个命令行窗口）：
   ```bash
   ngrok http 8000
   ```
7. **获取外网链接**：
   - ngrok会显示类似 `https://xxxx.ngrok.io` 的链接
   - 这个链接可以在任何地方访问

### 注意
- ⚠️ 免费版链接每次重启会变化
- ⚠️ 适合临时测试，不适合长期使用

## 常见问题

### Q1: GitHub Pages链接无法访问？
**检查**：
1. 确认仓库是Public（公开）
2. 确认Settings → Pages已正确配置
3. 等待1-2分钟让GitHub完成部署
4. 清除浏览器缓存后重试

### Q2: 如何自定义域名？
**GitHub Pages**：
1. 在Settings → Pages中添加自定义域名
2. 配置DNS记录（CNAME或A记录）

**Vercel/Netlify**：
1. 在项目设置中添加自定义域名
2. 按提示配置DNS

### Q3: 链接会过期吗？
- **GitHub Pages/Vercel/Netlify**：永久有效（只要仓库存在）
- **ngrok免费版**：每次重启会变化
